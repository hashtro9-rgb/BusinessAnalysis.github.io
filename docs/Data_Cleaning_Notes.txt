STEP-BY-STEP CLEANING PROCESS
================================================================================

STEP 1: INITIAL DATA INSPECTION (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Loaded raw dataset and examined structure
Command: SELECT * FROM dirty_business_dataset;
         CREATE TABLE DIRTY LIKE dirty_business_dataset;

Result:  5,060 rows × 30 columns identified
         Initial structure validated

Issues Found:
  • Multiple duplicate transaction IDs
  • Data type inconsistencies (text vs numeric)
  • Mixed case values in categorical fields
  • Inconsistent date formats

────────────────────────────────────────────────────────────────────────────

STEP 2: IDENTIFY DUPLICATES (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Find duplicate rows using ROW_NUMBER()

Code:    WITH CTE AS (
           SELECT *, ROW_NUMBER() OVER (PARTITION BY all_columns) AS ROW_NUM
           FROM DIRTY
         )
         SELECT * FROM CTE WHERE ROW_NUM > 1;

Result:  9 duplicate rows identified
         - Same TRANSACTION_ID, ORDER_DATE, COMPANY_NAME, etc.
         - Likely system migration or upload errors

────────────────────────────────────────────────────────────────────────────

STEP 3: STANDARDIZE CASING (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Convert all categorical text to UPPERCASE for consistency

Fields Updated:
  • COMPANY_NAME
  • INDUSTRY
  • COUNTRY
  • REGION
  • CUSTOMER_SEGMENT
  • CHANNEL
  • PRODUCT_NAME
  • PAYMENT_METHOD
  • TRANSACTION_STATUS
  • PRIORITY
  • CUSTOMER_SATISFACTION
  • REPEAT_CUSTOMER
  • NOTES
  • SALES_REP

Code:    UPDATE table_name
         SET COLUMN = UPPER(COLUMN)
         WHERE COLUMN IS NOT NULL;

Result:  Removed case-sensitivity issues
         Enabled proper grouping in analysis

────────────────────────────────────────────────────────────────────────────

STEP 4: STANDARDIZE COLUMN NAMES (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Rename all columns to UPPERCASE for consistency

Changes:
  Transaction_ID      → TRANSACTION_ID
  Order_Date          → ORDER_DATE
  Delivery_Date       → DELIVERY_DATE
  Company_Name        → COMPANY_NAME
  Revenue             → REVENUE
  (... 25 more columns)

Result:  Consistent naming convention across all 30 columns
         Easier to reference in queries and code

────────────────────────────────────────────────────────────────────────────

STEP 5: STANDARDIZE DATE FORMATS (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Convert all dates to ISO format (YYYY-MM-DD)

Found Formats:
  • YYYY-MM-DD (standard)
  • MM/DD/YYYY (US format)
  • DD-Mon-YYYY (text format)

Code:    UPDATE table_name
         SET ORDER_DATE = DATE_FORMAT(
           STR_TO_DATE(ORDER_DATE, '%d-%b-%Y'), '%Y-%m-%d')
         WHERE ORDER_DATE LIKE '%-%-%' AND LENGTH(ORDER_DATE) > 10;

Result:  All 4,647 dates now in consistent YYYY-MM-DD format
         Date range: 2020-01-12 to 2025-06-01
         Ready for sorting and calculations

────────────────────────────────────────────────────────────────────────────

STEP 6: REMOVE DUPLICATES (MySQL)
────────────────────────────────────────────────────────────────────────────
Action:  Delete rows where ROW_NUM > 1 (keeping first occurrence)

Code:    DELETE FROM table_name WHERE ROW_NUM > 1;

Result:  9 duplicate rows removed
         4,651 rows remaining

────────────────────────────────────────────────────────────────────────────

STEP 7: HANDLE NEGATIVE REVENUES (Python)
────────────────────────────────────────────────────────────────────────────
Action:  Remove rows with REVENUE < 0 (impossible business transactions)

Code:    df = df[df['REVENUE'] >= 0].copy()

Result:  74 rows removed (negative revenues)
         Range now: $161.87 to $6,800,156.92
         Mean: $1,530,054.80
         4,577 rows remaining

────────────────────────────────────────────────────────────────────────────

STEP 8: HANDLE FUTURE DATES (Python)
────────────────────────────────────────────────────────────────────────────
Action:  Fix impossible future dates (2027 and beyond)

Code:    df.loc[df['ORDER_DATE'].dt.year >= 2027, 'ORDER_DATE'] = \
         pd.Timestamp('2025-06-01')

Result:  16 rows with future dates adjusted
         All dates now within logical range
         Latest date: 2025-06-01

STEP 9: REMOVE FINAL DUPLICATES (Python)
────────────────────────────────────────────────────────────────────────────
Action:  Final deduplication by TRANSACTION_ID

Code:    df = df.drop_duplicates(subset=['TRANSACTION_ID'], keep='first')

Result:  9 duplicate TRANSACTION_IDs removed
